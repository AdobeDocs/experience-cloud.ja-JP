# Experience Rollouts コンソールにログインします {#log-in}

エクスペリエンスのロールアウトを開始するには、チームの検索または作成、アクセスの要求、コンソールへのログインという3つの手順を実行します。

## 始める前に {#before-you-begin}

エクスペリエンスのロールアウトは、**チーム**&#x200B;を中心に整理されています。 各チームは、1つ以上のアプリケーションを所有し、それらのアプリケーションの機能フラグを管理します。 ログインする前に、チームに所属する必要があります。

製品またはエンジニアリングのリードに問い合わせて、プロジェクトにチームが既に存在するかどうかを確認します。 その場合は、チーム管理者に正しい[&#x200B; ユーザーの役割](../teams/user-roles.md)を追加するように依頼します。 まだチームが存在しない場合は、[新しいチームを作成](create-a-new-team.md)の手順に従います。

## 利用申請 {#request-access}

参加するチームが決まったら、組織のアクセス管理システムからアクセスをリクエストします。 詳細な手順については、[&#x200B; アクセスを要求](request-access.md)を参照してください。

リクエストが承認されると、チーム管理者が付与した役割に関連付けられた権限が付与されます。

## コンソールにログインします {#log-in-steps}

アクセスが許可された後：

1. [https://experience.adobe.com/](https://experience.adobe.com/)に移動し、組織の資格情報でログインします。
2. アプリケーションスイッチャーから「**エクスペリエンスのロールアウト**」を選択します。
3. 適切な環境を選択します。テストには&#x200B;**ステージ**、ライブロールアウトには&#x200B;**実稼動**。 詳しくは、[環境の概要](environments-overview.md)を参照してください。

## ログイン後の最初の手順 {#first-steps}

ログイン後、アプリケーションがコンソールに表示されていることを確認します。 アプリケーションは、チーム管理者が管理します。 アプリケーションがリストにない場合は、チーム管理者に連絡してアプリケーションを追加してもらいます。 詳しくは、[&#x200B; アプリケーションの管理](../applications/manage-applications.md)を参照してください。

## 主な用語 {#terminology}

| 用語 | 定義 |
|---|---|
| **チーム** | アプリケーションを所有し、機能フラグを管理する自己管理グループ。 チームはフラットな構造をしており、ユーザーの役割や権限レベルも異なります。 |
| **アプリケーション** | 機能フラグで制御するアプリケーション。 各アプリケーションは、チームによって所有されます。 |
| **機能フラグ / 機能グループ / リリース** | 機能テストとリリース管理のためにエクスペリエンスロールアウトで作成されたアーティファクト。 |
