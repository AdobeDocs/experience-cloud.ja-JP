# 新しいチームを作成 {#create-a-new-team}

エクスペリエンスのロールアウトにプロジェクト用のチームが存在しない場合は、機能フラグの作成を開始する前に、チームをリクエストする必要があります。 チーム作成は、サポートリクエストを通じて処理します。

## 始める前に {#before-you-begin}

プロジェクトにチームが既に存在しないことを確認します。 製品またはエンジニアリングの責任者に確認するか、[ チームとその管理者](teams-and-admins.md)を確認してください。 チームが既に存在する場合は、代わりにチームへの参加をリクエストします。[ アクセスをリクエスト ](request-access.md)を参照してください。

## チームリクエストを送信 {#submit-request}

Experience Rollouts サポートに連絡して、新しいチームをリクエストしてください。 リクエストに次の情報を入力します。

| フィールド | 説明 |
|---|---|
| **チーム名** | チームの一意の名前。 |
| **ターゲティングモード** | チームが使用するターゲティングモード：[ ユーザーレベルのターゲティングまたは組織および環境レベルのターゲティング ](../../getting-started/experience-rollouts-modes.md)。 |
| **環境** | 必要な環境：ステージング、実稼動環境、またはその両方。 |
| **チーム管理者** | チーム管理者として機能するユーザーまたはユーザーの電子メールアドレス。 |

>[!NOTE]
>
>チーム管理者は、メンバーの管理とアクセスリクエストの承認を担当します。 1人の管理者が利用できない場合にアクセスのギャップを避けるために、少なくとも2人の管理者を指定します。

## チームの作成後 {#after-creation}

チームを編成すると、管理者はセルフサービス方式でメンバーの追加を開始できます。 新規メンバーはアクセス管理システムを通じてアクセスをリクエストし、管理者はそのリクエストを承認または却下します。

メンバーがチームに参加するための手順については、[ アクセスを要求](request-access.md)を参照してください。
