# 利用申請 {#request-access}

エクスペリエンスのロールアウトへのアクセスは、チームレベルで管理されます。 コンソールを使用するには、適切な役割を持つチームに追加する必要があります。

## 前提条件 {#prerequisites}

アクセスをリクエストする前に、次の点を確認してください。

* どのチームに参加すればいいかわかります。 製品またはエンジニアリングの責任者に確認するか、[ チームとその管理者](teams-and-admins.md)を参照して、チームとその管理者を特定します。
* アクセスが必要な環境（**ステージ**、**実稼動**、またはその両方）を把握できます。

## 利用申請 {#request-steps}

1. 組織のアクセス管理システムに移動します。
2. **エクスペリエンスロールアウト**&#x200B;を検索し、環境（ステージングまたは実稼動）に適したアプリケーションを選択します。
3. チーム名を検索し、要求している役割を選択します。
4. リクエストを送信します。

リクエストが送信されると、確認メールが届きます。 アクセス管理システムの&#x200B;**保留中**&#x200B;のリクエストで、リクエストのステータスを監視できます。

>[!NOTE]
>
>チームの&#x200B;**管理者**&#x200B;または&#x200B;**機能管理者**&#x200B;の役割を持つメンバーは、アクセス要求を承認または拒否できます。

## 利用可能な役割 {#roles}

割り当てられた役割によって、コンソールで実行できるアクションが決まります。 各役割とその権限の詳細については、[ ユーザーの役割](../teams/user-roles.md)を参照してください。

## アクセスが許可された後 {#after-access}

リクエストが承認されたら、Experience Rollouts コンソールにログインし、チームとアプリケーションが表示されていることを確認します。 次の手順については、[ コンソールにログイン ](log-in-to-the-console.md)するを参照してください。

## サポートが必要な場合 {#need-help}

参加するチームが不明な場合、またはプロジェクトにチームがまだ存在しない場合は、[新しいチームを作成](create-a-new-team.md)を参照するか、組織のExperience Rollouts管理者にお問い合わせください。
