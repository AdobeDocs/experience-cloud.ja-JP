# 機能、機能グループ、リリース {#features-feature-groups-releases}

エクスペリエンスのロールアウトには、機能のロールアウトを管理するための4つのアーティファクトが用意されています。 適切な手法を選定するには、導入範囲、関与するチーム数、必要なオーディエンスターゲティング機能などによって決まります。

## 4つのアーティファクト {#artifacts}

**機能フラグ**
最も原子的な単位。 1つのチームが所有する1つのアプリケーションの1つの機能を制御します。 定義されたオーディエンスに対して有効または無効にできます。

**機能グループ**
同じチームに属する機能フラグのコレクション。 複数のアプリケーションにまたがる複数のフラグを1つのチームで単一のユニットとして管理できます。

**チーム間の機能グループ**
複数のチームやアプリケーションをまたいで機能グループを拡張できます。 セルフサービスで、豊富なオーディエンス基準をサポートしていますが、キャッシュはサポートしていません。

**リリース**
複数のチームやアプリケーションをまたいで、大規模なロールアウトにも対応します。 認証トークンベースのオーディエンスターゲティングを使用します。 サーバーSDKでのキャッシュをサポートします。

## 比較 {#comparison}

| | 機能フラグ | 機能グループ | チーム間の機能グループ | リリース |
|---|---|---|---|---|
| オーディエンスの管理には&#x200B;**役割が必要です** | 製品リリース所有者 | 製品リリース所有者 | 機能管理者 | リリースマネージャー |
| **アプリケーション** | シングル | 複数（同じチーム） | 複数（クロスチーム） | 複数（クロスチーム） |
| **チーム** | シングル | シングル | チーム間 | チーム間 |
| **豊富なオーディエンス条件** | ✓ | ✓ | ✓ | 制限付き |
| **パーセンテージのロールアウト** | オーディエンス条件と組み合わせて利用すれば | オーディエンス条件と組み合わせて利用すれば | オーディエンス条件と組み合わせて利用すれば | 固定されたオーディエンス基準と組み合わせ |
| **A/B テスト** | ✓ | ✓ | ✗ | ✗ |
| **サーバーSDKでのキャッシュ** | デフォルトのオン/オフフラグのみ | キャッシュなし | キャッシュなし | ローカルにキャッシュされたすべてのリリース |
| **セルフサービス** | ✓ | ✓ | ✓ | サポートリクエストが必要 |
| **監査記録** | ✓ | ✓ | ✓ | ✓ |

## CDPの活用事例 {#when-to-use}

| シナリオ | 用途 |
|---|---|
| 単一の機能を単一のアプリケーションでテストまたはロールアウトする | **機能フラグ** |
| 同じチームの複数の機能を同時に公開する | **機能グループ** |
| 高度なターゲティングにより、様々なチームのアプリケーションをまたいで機能を調整する | **チーム間の機能グループ** |
| SDKレベルのキャッシュにより、複数のチームをまたいだ大規模なリリースを連携できます | **リリース** |
