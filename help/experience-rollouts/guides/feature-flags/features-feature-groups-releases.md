# 機能と機能グループ {#features-feature-groups}

エクスペリエンスロールアウトには、機能ロールアウトを管理するための2つのアーティファクトが用意されています。 適切なCMSを選定するには、導入範囲と機能の数によって異なります。

## 2つのアーティファクト {#artifacts}

**機能フラグ**
最も原子的な単位。 単一アプリケーションの単一の機能を制御します。 定義されたオーディエンスに対して有効または無効にできます。

**機能グループ**
同じチームに属する機能フラグのコレクション。 複数のアプリケーションにまたがる複数のフラグを1つのユニットとして管理できます。

## 比較 {#comparison}

| | 機能フラグ | 機能グループ |
|---|---|---|
| オーディエンスの管理には&#x200B;**役割が必要です** | 製品リリース所有者 | 製品リリース所有者 |
| **アプリケーション** | シングル | 複数（同じチーム） |
| **豊富なオーディエンス条件** | ✓ | ✓ |
| **パーセンテージのロールアウト** | オーディエンス条件と組み合わせて利用すれば | オーディエンス条件と組み合わせて利用すれば |
| **A/B テスト** | ✓ | ✓ |
| **セルフサービス** | ✓ | ✓ |
| **監査記録** | ✓ | ✓ |

## CDPの活用事例 {#when-to-use}

| シナリオ | 用途 |
|---|---|
| 単一の機能を単一のアプリケーションでテストまたはロールアウトする | **機能フラグ** |
| 同じチームの複数の機能を同時に公開する | **機能グループ** |
