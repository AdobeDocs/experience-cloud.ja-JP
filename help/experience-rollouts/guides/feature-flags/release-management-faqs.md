# リリース管理に関するFAQ {#release-management-faqs}

## リリースをリクエストするにはどうすればよいですか？ {#request-release}

Experience Rollouts サポートに連絡してリリースをリクエストしてください。 チーム名、アプリケーションの詳細、ターゲットオーディエンス、必要なロールアウトタイムラインを提供します。

## リリースでサポートされるオーディエンス基準は何ですか？ {#audience-criteria}

リリースでは、次のオーディエンスルールがサポートされています。

* ID タイプ（アカウントタイプ）
* 国
* ユーザーID （GUID）
* メールアドレス（個人または一括リスト）
* メールドメイン
* クライアント IP
* 割合（全ユーザー）
* 割合（認証済みユーザーのみ）

ネストされた条件を含むAND/OR ロジックを使用して、複数のルールを組み合わせることができます。

## リリースにアプリケーションを追加するにはどうすればよいですか？ {#onboard-application}

リリースを作成したら、コンソールでリリースを開き、アプリケーションをリリース設定に追加します。 各アプリケーションの製品リリース所有者は、関連する機能フラグを追加できます。

## 条件を満たす必要のあるユーザーに対して、機能が返されません。 トラブルシューティング方法を教えてください。 {#troubleshoot}

デバッグするには、次の手順に従います。

1. **リリースの状態を確認してください。** 機能を提供するには、リリースが&#x200B;**公開済み**&#x200B;または&#x200B;**完全ロールアウト**&#x200B;状態である必要があります。
2. **アプリケーションと機能フラグを確認します。** 機能フラグが正しいアプリケーション用に作成され、アプリケーションが正しいリリースに関連付けられていることを確認します。
3. **機能フラグが有効になっていることを確認します。** リリースがアクティブな場合でも、無効なフラグは提供されません。
4. **オーディエンス条件を確認します。** ユーザーが、オーディエンスルールで定義されているすべての条件を満たしていることを確認します。 フィーチャーが関連付けられている特定のリリースの基準を再確認します。
