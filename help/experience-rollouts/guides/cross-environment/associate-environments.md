# 環境をアプリケーションに関連付ける {#associate-environments}

環境間でアプリケーションインスタンスをリンクすることで、環境間の可視化とワークフローのインポートが可能になります。 これを設定するには、**管理者**&#x200B;の役割が必要です。 役割を確認または更新する必要がある場合は、チーム管理者にお問い合わせください。

これが便利な理由の背景については、[環境をまたいだ概念](cross-environment-concept.md)を参照してください。

## 手順1：アプリケーション設定を開く {#step-1}

1. Experience Rollouts コンソールにログインします。
2. **管理者/ アプリケーション**&#x200B;に移動します。
3. 新しいアプリケーションをオンボーディングするには、**新しいアプリケーション**&#x200B;を選択するか、既存のアプリケーションを選択して編集します。

## 手順2：関連する環境の設定セクション {#step-2}

アプリケーションフォームで、「**関連する環境**」セクションまでスクロールします。 このセクションの各行は、アプリケーションのデプロイメント環境のいずれかを定義します。 環境ごとに、次のフィールドを設定します。

| フィールド | 説明 |
|---|---|
| **環境** | リストから標準的な環境識別子（QA1、QA2、STG1、STG2、PROD1、PROD2）を選択します。 これにより、環境が実稼動環境であるか非実稼動環境であるかをエクスペリエンスロールアウトに伝え、コンソールで使用されるカラーコーディングを決定します。 |
| **環境ラベル** | チームがこの環境を参照する場合の、この環境のカスタム名（例：`Dev`、`Staging`、または`Production`） これはフリーフォームであり、標準の識別子と一致する必要はありません。 |
| **エクスペリエンスのロールアウト環境** | このアプリケーションインスタンスがホストされているプラットフォーム環境（ステージングまたは実稼動）。 現在のコンソールに基づいて事前入力されます。 |
| **アプリケーション ID** | この環境のアプリケーションインスタンスのクライアント ID。 ドロップダウンからアプリケーションを選択すると、自動入力されます。 |

## 手順3：環境間でアプリケーションインスタンスをリンクする {#step-3}

環境間でインスタンスをリンクするには、追加環境ごとに次の手順に従います。

1. ターゲット環境でアプリケーションインスタンスを作成します（例えば、ステージコンソールでQA アプリを作成します）。
2. 設定しているアプリケーションに戻り、**関連する環境**&#x200B;に新しい行を追加します。
3. **アプリケーション ID** ドロップダウンから、作成したばかりのアプリケーションインスタンスを選択します。 環境タグとラベルは自動的に入力されます。
4. 保存する&#x200B;**更新**&#x200B;を選択します。

同じアプリケーションのQA、ステージ、実稼動インスタンスをリンクするなど、リンクする各環境に対して、このプロセスを繰り返します。

## 重要な注意事項 {#important-notes}

* QA環境とステージ環境は、Experience Rollouts Stage環境または実稼動プラットフォーム環境でホストできます。
* 実稼動環境（PROD1、PROD2）は、Experience Rollouts実稼動環境でのみホストできます。
* 上位の環境からのみ下位の環境にリンクできます。 例えば、実稼動環境からステージインスタンスまたはQA インスタンスにリンクできますが、その逆は読み取り専用です。
