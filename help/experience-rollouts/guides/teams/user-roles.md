# ユーザーロール {#user-roles}

エクスペリエンスのロールアウトには5つの役割があります。 それぞれの役割は、特定の責任セットに対して設計されています。 デフォルトでは、役割は相互に排他的です。管理者ロールを持つメンバーには、製品リリース所有者の権限は自動的には付与されません。 広範なアクセスが必要な場合は、メンバーに複数の役割を割り当てます。

## 利用可能な役割 {#roles}

| 役割 | 責任 |
|---|---|
| **Admin** | コンソールにアプリケーションをオンボーディングし、チームメンバーを管理し、アクセス要求を承認または拒否します。 チームが機能フラグの使用を開始する前に必要です。 |
| **製品リリース所有者** | アプリケーションの機能フラグと機能グループを作成および更新します。 オーディエンスを機能フラグにリンクして、外部ユーザーが機能を利用できるようにすることができます。 |
| **開発者** | サンドボックス化されたコンテキストで動作し、他のユーザーに影響を与えることなく、機能フラグの背後にある機能をテストします。 機能フラグをステージングまたは実稼動環境のプライベートオーディエンス（特定のユーザーIDやメールアドレスなど）に公開できます。 |
| **機能管理者** | チーム間の機能グループを作成および管理します。 この役割は、異なるチームが所有するアプリケーションをまたいでフラグを調整する場合に使用します。 |
| **リリースマネージャー** | マルチチーム、マルチアプリケーションのリリースを管理し、リリースの受信者を定義するオーディエンスルールを更新します。 |

## 適切な役割の選定 {#choosing}

役割を割り当てるには、次のガイダンスを使用します。

* **アプリケーションの機能フラグを追加または更新しています** → **製品リリース所有者**
* **他のユーザーに影響を与えることなく、機能をプライベートでテストする** → **開発者**
* **チーム間の機能グループの管理** → **機能管理者**
* **複数のチームとアプリケーションにわたるリリースの管理** → **リリースマネージャー**
* **アプリケーションのオンボーディングとチームメンバーシップの管理** → **管理者**

>[!NOTE]
>
>メンバーは複数の役割を保持できます。 例えば、チームのアプリケーションの機能をテストし、管理するエンジニアは、**開発者**&#x200B;と&#x200B;**管理者**&#x200B;の両方の役割を持っている必要があります。

## 権限エラー {#permissions-error}

機能またはグループを更新しようとしたときに「十分な権限がありません」というエラーが発生した場合は、**製品リリース所有者**&#x200B;の役割が必要です。 この役割を追加するには、チーム管理者にお問い合わせください。
