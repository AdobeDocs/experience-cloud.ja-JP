# デスクトップアプリケーション {#desktop-applications}

デスクトップアプリケーションは、REST APIを直接呼び出すことで、エクスペリエンスのロールアウトと統合されます。 デスクトップクライアントに&#x200B;**機能API V2**&#x200B;を使用します。

完全なAPI リファレンスについては、このガイドの機能API セクションの&#x200B;**GET機能API V2**&#x200B;を参照してください。

## 統合要件 {#requirements}

デスクトップクライアントは次の操作を行う必要があります。

* **API応答で返されたTTL値**&#x200B;を尊重します。 TTLは、クライアントがAPIを再度呼び出す前に機能フラグデータをキャッシュする時間を定義します。 この動作を検証するテストケースを実装して、機能フラグが提供されていないときに古いアプリケーションバージョンが正しく休止状態に入るようにします。
* **HTTP エラーシナリオを正常に処理**。 フォールバック機能セットを組み込むことで、APIが一時的に利用できない場合でもアプリケーションが機能し続けることができます。
* **上記のTTLとエラー処理の両方の要件をカバーする詳細な統合テスト プラン**&#x200B;を維持します。

## アプリケーション ID {#app-id}

デスクトップクライアントは、APIを呼び出す際に、標準クライアント IDの代わりに&#x200B;**製品コードと製品バージョン**&#x200B;をアプリケーション識別子として使用できます。
