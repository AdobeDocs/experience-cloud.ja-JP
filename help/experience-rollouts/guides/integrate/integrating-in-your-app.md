# アプリにエクスペリエンスのロールアウトを統合する {#integrate}

このセクションでは、すべてのExperience Rollouts クライアントの統合ガイドラインを、アプリケーションタイプ別に整理して提供します。

## 統合パスの選択 {#choose}

アプリケーションの種類に合ったガイドを選択します。

1. [ スタートアップガイド ](startup-guide.md) – すべての統合手順の概要については、ここから始めてください
2. [ デスクトップアプリケーション ](desktop-applications.md) — デスクトップアプリケーション用のREST API統合
3. [ モバイルアプリケーション ](mobile-applications.md) — モバイルアプリ用REST API統合
4. [Web アプリケーション ](web-applications.md) — web プロパティ用REST API統合
5. [Web サービス ](web-services.md) — バックエンドサービス用のサーバーサイド SDK統合
6. [SDK](sdks.md) — SDK アーキテクチャ、前提条件、使用可能なSDK
7. [統合手順](integration-steps.md) – 詳細なステップバイステップ統合手順

## アプリケーションタイプの概要 {#overview}

| アプリケーションタイプ | 連携の強化 |
|---|---|
| **Web サービス / バックエンド** | Java SDKまたはNode.js SDK |
| **Web アプリケーション** | REST API – 機能API V3 |
| **モバイルアプリケーション** | REST API – 機能API V3 |
| **デスクトップアプリケーション** | REST API – 機能API V2 |
