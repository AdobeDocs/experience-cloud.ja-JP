# オーディエンスルールでのコンテキストの使用 {#context-in-audience-rules}

コンテキスト変数は、実行時にクライアントアプリケーションによって提供される値です。 これにより、ユーザーのアクティブな言語、デバイスの種類、アプリケーションの状態など、ユーザーの永続的なプロファイルの一部ではない動的なセッションレベルの情報にもとづいて、ユーザーをターゲットにすることができます。

コンテキスト変数は、web、デスクトップおよびモバイルクライアントに関連しています。

## コンテキスト変数の仕組み {#how-context-works}

このアプリケーションは、機能フラグを評価する際に、エクスペリエンスロールアウトにコンテキスト変数を渡します。 これらの値をチェックするルールをコンソールで定義し、プラットフォームは評価時にそれらを使用して、ユーザーが適格かどうかを判断します。

オーディエンスルールの&#x200B;**プロファイル**&#x200B;と&#x200B;**コンテキスト**&#x200B;の両方のセクションで条件を定義すると、すべてのプロファイル条件がAND ロジックを使用してすべてのコンテキスト条件と組み合わされます。

## コンテキスト変数タイプ {#variable-types}

### 定義済み（リスト）タイプ {#predefined-type}

言語や国のリストなど、許可される値の固定セットを持つコンテキスト変数。

使用可能な演算子：**Is**、**Is Not equal to**、**Exists**、**In**

### 整数型 {#integer-type}

数値を保持するコンテキスト変数。

使用可能な演算子：**1}より大きい、** 3}以上、**は**、**より小さい**、**より小さい**&#x200B;です&#x200B;****

### 文字列タイプ {#string-type}

自由形式のテキスト値を保持するコンテキスト変数。

使用可能な演算子：**Is**、**Is not equal to**

## コンテキスト変数の追加 {#adding-context-variable}

オーディエンスルールにコンテキスト変数を追加するには：

1. コンソールで機能フラグまたは機能グループを開きます。
2. 「**オーディエンス**」タブに移動します。
3. **Context**&#x200B;で、新しい条件を追加します。
4. コンテキスト変数、演算子、値を選択します。

必要なコンテキスト変数がリストに表示されない場合は、コンソールのコンテキスト変数管理セクションからセルフサービス方式で新しいコンテキスト変数を作成できます。

>[!NOTE]
>
>コンテキスト変数がオーディエンス条件に含まれている場合、コンテキスト値は実行時に決定され、事前に予測できないため、**オーディエンス計算ツール**&#x200B;は推定カウントを返しません。
