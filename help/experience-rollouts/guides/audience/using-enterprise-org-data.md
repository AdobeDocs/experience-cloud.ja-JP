# オーディエンスルールでの企業組織データの使用 {#enterprise-org-data}

エクスペリエンスのロールアウトは、エンタープライズ組織（組織） IDに基づいてユーザーをターゲティングすることをサポートします。 これにより、特定の顧客組織に機能をロールアウトしてから、広く利用できるようにします。

## オーディエンスルールへの組織の追加 {#adding-org-rule}

エンタープライズ組織のターゲティングは、**オーディエンスルール/プロファイル/エンタープライズ**&#x200B;の「**オーディエンス**」タブで利用できます。

次の2つの組織タイプがサポートされています。

### DME組織 {#dme-orgs}

DME組織は、組織IDによって識別されます。 ルールを作成する場合は、組織IDのみを入力します。認証ソースは含めないでください。

### DMA組織 {#dma-orgs}

DMA組織は、組織IDを`XXXXXXXXXXXXXXXXXXXXXXXX@ADOBEORG`形式で使用します。 DMA組織IDを入力する場合：

* `@ADOBEORG`のサフィックスを含む完全な組織IDを使用してください。
* すべての大文字で`ADOBEORG`と入力します。

**例：** `57F22B5D5A5F83AE0A495E6E@ADOBEORG`

## 重要な注意事項 {#important-notes}

* 組織ベースのターゲティングは、ユーザーの認証済みセッションに関連付けられた組織に対して評価されます。 テスト時に、ユーザーが正しい組織にサインインしていることを確認します。
* 見つけると思われる組織がオーディエンスの条件に表示されない場合、または特定の組織の機能が期待どおりに返されない場合は、Experience Rollouts サポートにお問い合わせください。
* ステージング環境と実稼動環境では、どの組織を使用できるかが異なる場合があります。 本番環境にプロモートする前に、ステージでオーディエンスルールをテストします。
