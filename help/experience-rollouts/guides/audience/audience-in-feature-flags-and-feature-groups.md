# 機能フラグと機能グループのオーディエンス {#audience-overview}

エクスペリエンスのロールアウトでは、機能フラグと機能グループに対してリッチなオーディエンスターゲティングを提供します。 プロファイル属性、コンテキスト変数、オーディエンスセグメントを組み合わせて、どのユーザーが機能を受け取るかを正確に制御できます。

## プロファイルルール {#profile-rules}

プロファイルルールにより、国、メールドメイン、アカウントタイプ、ユーザーIDなどの属性にもとづいてユーザーをターゲティングできます。 AND/OR演算子を使用して複数の属性を組み合わせ、複雑なターゲティングシナリオ用のネストされたロジックを構築できます。

プロファイルルールを追加するには、機能フラグまたは機能グループの「**オーディエンス**」タブに移動し、**プロファイル**&#x200B;の下に条件を追加します。

一連のオーディエンスルールを再利用可能な&#x200B;**オーディエンス**&#x200B;として保存し、複数のフラグやグループで使用できます。

コンテキストベースのターゲティング（例えば、ユーザーのアクティブな言語またはクライアントタイプによるターゲティング）については、[ オーディエンスルールでコンテキストを使用](using-context-in-audience-rules.md)を参照してください。

## セグメント {#segments}

機能フラグと機能グループは、オーディエンスセグメントをサポートします。 セグメントを活用すると、以下のことが可能になります。

* 複数の機能フラグやグループに対して、単一の事前定義済みオーディエンス定義を使用できます
* **イベントベースの条件**&#x200B;をオーディエンスターゲティングに含めます。プロファイルルールだけでは不可能です

セグメントを使用するには、「**オーディエンス**」タブに移動し、リストから1つ以上のセグメントを選択します。 AND/OR演算子を使用して複数のセグメントを組み合わせ、プロファイルまたはコンテキストフィルターを追加できます。

>[!NOTE]
>
>オーディエンスセグメントを作成するには、**テストマネージャー**&#x200B;以上の役割が必要です。

## オーディエンス計算機 {#audience-calculator}

オーディエンス条件を定義したら、下部のバーで「**計算**」を選択して、適格なユーザーの推計数を取得します。 これにより、機能をアクティブ化する前に、ターゲティングのリーチを検証できます。

>[!NOTE]
>
>オーディエンス計算機は、コンテキスト変数が条件に含まれている場合、コンテキスト値は実行時に決定され、事前に予測できないため、値を返しません。
