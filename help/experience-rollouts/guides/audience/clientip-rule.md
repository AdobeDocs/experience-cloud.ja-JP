# クライアント IP コンテキスト変数を使用したオーディエンスルール {#clientip-rule}

`clientip` コンテキスト変数を使用すると、クライアント IP アドレスに基づいてユーザーをターゲットにできます。 これは、ユーザーがアプリケーションにアクセスするネットワークに基づいて機能を制限または有効にする場合（例えば、企業ネットワーク上のユーザーへの早期アクセスを制限する場合など）に便利です。

## サポートされる入力タイプ {#input-types}

`clientip` コンテキスト変数は、次の3種類の入力値をサポートしています。

| 入力タイプ | 説明 | サポートされている演算子 |
|---|---|---|
| **IP アドレス** | 単一のIPv4またはIPv6 アドレス | 次の値になります： |
| **CIDR ブロック** | CIDR表記法のIP アドレスの範囲（例：`192.168.1.0/24`） | 次と等しくありません |
| **ネットワーク範囲** | プラットフォームが維持する、事前定義済みの名前付きネットワーク範囲 | 次の一部である |

## クライアント IP ルールの追加 {#adding-rule}

1. 機能フラグ、機能グループ、またはチーム間の機能グループをコンソールで開きます。
2. 「**オーディエンス**」タブに移動します。
3. **Context**&#x200B;で、**clientip**&#x200B;変数を使用して条件を追加します。
4. オペレーターを選択し、値（IP アドレス、CIDR ブロック、または定義済みのネットワーク範囲の選択）を入力します。
