# SDK ベンチマーク {#sdk-benchmarking}

エクスペリエンスのロールアウト Java SDKは、SDKがインフラストラクチャ内で実行される場合に想定されるリソースと応答時間を信頼できる方法で見積もるようにベンチマークされています。

## テスト環境 {#test-environment}

ベンチマーキングは、次の固定設定を使用してSpring Boot アプリケーションで実行されました。

* **CPU コア：** 8
* **JVM メモリ：** 4096 MB

## テストの仮定 {#test-assumptions}

以下の条件は、すべてのベンチマーク実行で一定に保たれました。

* テスト済みの機能フラグ：8～128
* 機能フラグごとのコンテキスト変数：2 （タイプ `STRING`、長さ36文字）
* 平均負荷：毎分15,000 リクエスト（RPM）
* アクティブスレッド：100

## 結果 {#results}

次の表は、機能フラグの数が増えると、`getFeatures()`呼び出しの99.99 パーセンタイル応答時間を示しています。

| 機能フラグの数 | フラグごとのコンテキスト変数 | 応答時間（ミリ秒、p99.99） |
|---|---|---|
| 8 | 2 | &lt; 0.5 |
| 16 | 2 | &lt; 0.5 |
| 32 | 2 | &lt; 0.5 |
| 64 | 2 | &lt; 1 |
| 128 | 2 | &lt; 1 |

## 結果の解釈 {#interpretation}

上記のベンチマークは、記載されている特定の試験条件での性能を表します。 SDKはアプリケーションのインフラストラクチャ内で動作するため、実際の応答時間は、環境に固有の要因によって異なります。

* 利用可能なCPUとコア数
* JVM ヒープサイズとガベージコレクションの動作
* スレッドの可用性とコンテキストの切り替え
* リクエスト時の現在のシステム負荷

これらの数値は、本番環境のパフォーマンス目標を保証するものではなく、計画のベースラインとして使用します。
