# 複数の機能を制御する機能グループ {#feature-groups}

[機能フラグ ](what-is-a-feature-flag.md)は、1つの機能を制御します。 複数の関連する機能フラグを一緒に管理し、同じオーディエンスに確実に届ける必要がある場合は、**機能グループ**&#x200B;を使用します。

## 機能グループの役割 {#what-it-does}

機能グループは、複数の機能フラグをバンドルし、1つのオーディエンスルールをすべての機能に一度に適用できます。 これは、単一のユーザー対応機能が複数のアプリケーションやサービスにまたがる場合に便利です。

たとえば、デスクトップアプリケーション、モバイルアプリ、バックエンドサービスをまたいで変更を加えるコラボレーション機能について考えてみましょう。 3つのアプリケーションすべてのフラグを含む機能グループを作成することで、1%のユーザーに対して機能を開くことができ、それらのユーザーが3つのサーフェス全体で同時に一貫性のあるエクスペリエンスを確認できます。

## アプリケーション間のグループ化 {#cross-application}

機能グループは、フラグがExperience Rolloutsの&#x200B;**same team**&#x200B;に属している限り、アプリケーション間の機能管理をサポートします。 1つのチームが複数のアプリケーションを所有できるため、それらのアプリケーション間の関連フラグをグループ化できます。

## 機能グループとリリース {#vs-releases}

| | 機能グループ | リリース |
|---|---|---|
| 範囲 | ひとつのチームで管理 | 統合および標準化するのに役立ちます |
| ユースケース | チーム内のフラグを調整する | 大規模なマルチチームのローンチ調整 |
| 必要な権限 | 導入にとどまっている | より高い（リリースマネージャー） |

グループ化する機能フラグが別のチームが所有するアプリケーションに属している場合は、機能グループの代わりに[ リリース ](release-management.md)を使用します。
